Video Encoding Guidelines

Skip to end of metadata
  • Created by Bill Reid, last modified by Robert Burdick on Feb 01, 2016
Go to start of metadata

The following guidelines for encoding video content are provided to allow maximum streaming playback efficiency and quality on a Roku Streaming Media Player, within the following constraints:

  • the video codecs supported by the Roku Player
  • the original and available source content formats
  • the adaptive bitrate streaming formats supported by the Roku Player to compensate for varying network speeds
  • Roku does not support interlaced video

Before starting your encodings, you will need to understand the constraints of the Roku Streaming Media Player, and any encoding changes required for certain sources of video content.  You are encouraged to encode your content at multiple bit rates as the Roku Player can take advantage of these different encodings by measuring the available bandwidth, and selecting the stream that requires a bandwidth less than the measured bandwidth. Anytime the Roku Player needs to buffer more data, it will pick the appropriate stream again. The H.264 encodings are the most efficient encoding on the Roku Player. H.264 is used in the .mp4 container as well as HLS, Smooth (ISM), and DASH. It is often combined with AAC-LC or AC-3 audio in the MP4 container. HLS requires AAC-LC audio.

Adaptive Bitrate Formats

Because network speeds can vary over time, Roku Players support several methods to provide the highest-quality video possible at any given time while avoiding streaming interruptions. All of these methods involve providing several levels of encoding quality for the video content which can be automatically selected by the Roku Player as the network speed varies.

There are two primary methods of encoding adaptive bitrate content for the Roku Player. The Roku Player offers native multi-file support using progressive download as the default method. This method supports the encoding of different bitrates for a piece of content, and the passing of URLs referencing the different encoded video files for each bitrate to stream to the Roku Player. Whenever the Roku Player detects a streaming underrun due to a lower network speed, it can switch to a lower bitrate encoding, and continue playing at the current timestamp after buffering the required amount of data for that bitrate.

The second encoding method uses widely-used standard formats for adapative bitrate switching. The following standard adaptive bitrate switching formats are supported by the Roku Player:

  • HLS (HTTP Live Streaming)
  • Smooth Streaming
  • DASH (Dynamic Adaptive Streaming over HTTP)

Each of these formats can proactively change the bitrate it plays on any "segment boundary" (at specified intervals) without a streaming underrun or buffering event. We'll try to give you a sense of tradeoffs available and where different types of content might fit.

Live Video

Roku supports live video streaming in the HLS and Smooth Streaming formats.

Roku 4 Streaming Media Player Considerations

Support is now included for the Roku 4 Streaming Media Player. This Roku Player features greatly enhanced video resolution output up to 2160p (ultra-high definition, or UHD). This Roku Player also allows the creation and use of user interfaces with 1080p (full high-definition, or FHD) resolution. The Roku 4 Streaming Media Player includes a much more powerful quad core ARM processor and decoding support for the HEVC (high-efficiency video code) codec to allow efficient streaming of 2160p video content.

Compatibility with Existing Channels

All existing channels and applications are fully compatible with the Roku 4 Player. Special considerations apply if you want to stream 2160p content as part of your channel or application, or if you want to create and use a 1080p user interface for your channel or application.

UHD Video Content Streaming Support and Recommendations

The following are the SDK changes and additions to support 2160p video content streaming:

  • GetVideoMode() method returns strings for 2160p video modes
  • CanDecodeVideo() method
  • CanDecodeAudio() method
  • GetDrmInfo() method
  • GetDisplayProperties() method

The following are the recommendations for streaming 2160p video content using the Roku 4 Streaming Media Player.

  • Always provide AAC 2-channel stereo as an alternate audio to surround sound
  • DASH is the preferred adaptive streaming format
  • HEVC is the preferred video codec
  • The recommended bit rates are 8, 10, 15, and 20Mbps
  • The recommended adaptive streaming segment sizes are 2.5, 3.3, or 5 seconds
  • It is recommended the segment sizes be constant and the same for all bit rates for the best adaptive switching and trick play (with 10 second BIF files)
  • It is recommended to provide all recommended bit rates for FHD, HD, and SD in the same DASH stream

The following are the supported and recommended DASH encoding considerations. 

  • Supported Profiles
    • ISO Base Media File Format only
    • VOD only
  • Signaling
    • Supported video formats: AVC, HEVC
    • Supported audio formats: AAC, EAC-3
    • Audio and Video Representations are carried in separate AdaptationSets
    • MPD must contain either AVC or HEVC, not both
    • SegmentTemplate tag must be present in MPD or Segment Index box to be present in the MP4 file
  • Segment Size and Alignment
    • Duration in SegmentTemplate must be same as key frame interval
    • When encoding a Segment Index box
      • Roku recommends key frame interval of 10 seconds
      • All Representations must be encoded with the same key frame interval
    • If SegmentTemplate tag is encoded, all Representations of a certain component must use the same SegmentTemplate

    • All encoded segments must start with an IDR frame

    • SegmentTemplate must use segment number

  • Only PlayReady encryption is supported
  • Lowest and Highest Bitrates
    • Roku recommends the user have at least a 1.5Mbps connection
    • Some customers will have low bitrate connections around 1Mbps or experience intermittent network load that causes drops below Roku recommended network speeds
    • Roku recommends encoding a low bitrate stream around 600Kbps
