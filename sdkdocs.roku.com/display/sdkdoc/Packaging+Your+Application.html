Packaging Your Application

Skip to end of metadata
  • Created by Bill Reid, last modified on Nov 02, 2015
Go to start of metadata

There are a few simple steps required to package your application before uploading to the Roku Channel Store. This process creates a secure version of your application suitable for general release. This secure version is called a package. The packaging process uses cryptographic hardware on the player during the process, so you must have a player available to package an application.

The basic steps in creating an application package are:

  1. Side-load your application onto a Roku device (see Application Installer Page in the Developer Guide).
  2. Run the genkey utility to generate a key (see Run the genkey Utility). This step only needs to be done once.
  3. Run the package utility to generate the package (see Run the package Utility).
  4. Download the package from the Roku device to your computer (see Downloading the Package).

Run the genkey Utility

To run the Package Utility, you must first generate a key by running the console utility genkey. The genkey utility will generate three things:

  • A key to encrypt and sign your application. This key remains hidden and is never seen by you or anyone else.
  • A password to use the key. The key cannot be used unless you have this password.
  • A developer ID. This is a non-confidential token that identifies which key was used to encrypt and sign the application.

Whenever you update your application, you should reuse the same key.  This can only be done if you have the original password, so it is important to keep the password somewhere safe even after you have finished packaging your application.

To run genkey, you must have a telnet client.  A telnet client is available for all modern operating systems.  It is often simply called telnet, which is what we shall use in the examples below.

  1. On the Roku home screen, select Settings, then About. Note the IP address of the Roku player.
     
  2. From a computer, run your telnet application, using the IP address found in step 1 and port number 8080. For example, if the IP address were 192.168.1.45, you would type:
    telnet  192.168.1.45  8080
     
  3. In the telnet console, you should see your Roku's serial number and a prompt:
    18C215080317
    > _
     
  4. Press Enter a couple of times. The prompt should repeat.
     
  5. Type genkey and press Enter. You will see a number of dots print, then after a while it will print a password and developer ID:
    > genkey
    .....................................................
    Password: qUtgw1sjHZQabCaUqPDg3g==
    DevID: 671b0ea1dbffcc10f3dfc931718351cdecfacc0f
    > _
     

  6. Make a record of the password and developer ID so that you will have it after the telnet session closes. You will enter the password next when you run the Package Utility.

The key remains stored in the Roku player and can be used to create any number of packages. That is, you only need to run genkey once. The key is erased by a factory reset or by running genkey again. An erased key can be restored by rekeying the player as described in Rekeying. However a lost password cannot be recovered.

Run the package Utility

The package utility is accessed through the Developer Web Page on the Roku player. The Developer Web Page is disabled by default, but must be enabled in order to side-load your application. The instructions to enable the page are described under Enabling Development Mode in the Developer Guide but if you've gotten this far, it should already be enabled.

To run the packager, follow these steps:

  1. On your computer, open a browser and enter the IP address of your Roku player. This is the same address used in the first step of genkey.
     
  2. This will open the Development Application Installer page, showing that your application is installed.
     


    If instead you see a "No Development Application Installed" message like this, it means you have not successfully side-loaded your application on this Roku player.
     

     
  3. Click on the "Packager" link at the top of the page.  This will open the Application Packager page. 
     
    If you see a "no dev key" message instead of the packager link like this, it means you have not successfully run genkey on this Roku player.
     

     
  4. At the bottom of the Packager page, enter your application's name and version number in the first box (e.g. "MyApp/1.0"). 
     
  5. In the second box, enter the password obtained from genkey.
     

     
  6. Click the Package button.  After a few moments, the page should refresh.
     
  7. In the Currently Packaged Application section, there should now be a link to download the package.
     

     
  8. After your channel is packaged, you can also inspect the details of the package. This is useful for confirming the creation date of the package as well as the devid used to sign the package. Especially for channel updates, checking the devid is a helpful test to guarantee that the devid is not different from your current channel version in the Roku Channel Store. Changing the devid will result in any persistent data stored in the registry being lost.
     
    To inspect the package, click Utilities, then click the Inspect button. You will see output similar to that in the image below.
     

Downloading the Package

After completing the package process, simply click on the link under "Currently Packaged Application" to download the package file to your computer. This package file can then be uploaded to the Channel Store.

The name of the downloaded package will be

Pxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.pkg

where the xxxx are 32 hex digits.  These digits are the MD5 checksum of the contents of the file.  Thus, it is possible to verify that the file contents have not been corrupted by typing this at a bash shell:

 m1=${filename:1:32}
 m2=$(md5sum $filename | sed 's/ .*//')
 if [ $m1 = $m2 ]; then echo OK; else echo CORRUPT; fi

This is for convenience only; it is not necessary to verify the file in this way.   Also, you may rename the file if you wish before uploading it to the Channel Store.

Rekeying

In some circumstances it may be convenient to install the same key on more than one Roku box, or to restore a key onto a Roku box after it has been erased. This can be done if you have a package file which was created using the desired key, and the password for the key.  The process of copying a key from a package back to a Roku player is called rekeying. To rekey a Roku player, follow these steps:

  1. On your computer, open a browser and enter the IP address of your Roku player.  This will open the Development Application Installer page.
     
  2. Click on the Utilities link at the top of the page.  This will open the Application Package Utilities page.
     

     
  3. Click the browse button and select the package file which was created using the key that you wish to copy.
     
  4. Enter the password for that key in the Password box.
     
  5. Click the Rekey button.

This will install the key onto the Roku player.  To generate a new package using this key, run the Package Utility as described in Package Utility.  You should not run genkey again, because that will generate a new key which will overwrite the rekeyed key.

Brief Overview of the Package Security Model

The Roku Streaming Player is designed to be a general purpose streaming video player capable of supporting multiple content providers and applications through the Channel Store. The Security Model has provisions in place to prevent unauthorized access to content, services or any other application specific data. The Roku firmware was designed to provide safe, "sandboxed" access to platform services through a virtual machine environment. The "sandbox" provides:

  • Restricted access to platform services
  • No access to extend or modify binaries
  • No access to audio/video data
  • Restricted access to other developers application data
  • No access to data from applications signed with a different key
  • No access to log, console or other output
  • No access to source code for other applications

Each application has access to a secure system registry for storage of application specific data. The registry is encrypted for safe storage of secrets and can only be read by applications signed with the same key. The registry is unique to a specific developer context. In general, each application maintains its own unique registry, but if a family of applications needed to share registration information this is also possible by using the same key for all applications.

When a developer performs a genkey on their Roku Streaming Player, they generate an RSA key pair. When they package their application, they sign and encrypt the application with the private key genkey produced. The package signature also creates an association to a secure registry. The firmware enforces access to the registry based on the key that signed the package.

The developer can create multiple applications that share a common registry by signing all of the applications with the same key. If the private key (or its password) is lost, new versions of the application will be unable to access the registry. Since the private key is stored in the package, but protected with AES encryption via the password, it can always be recovered from a previous package as long as the password is remembered. So it is of utmost importance to safely and securely store that password and package containing the private key. With the password and a package containing the corresponding keyset from a genkey, any Roku Streaming Player can be "Rekeyed" to enable signing of packages with the same key that was stored in a previous package.

Consequences of the Package Security Model

  • Applications can share data via registry entries if and only if the applications are packaged with same key
  • New versions of applications will have to re-link if they are packaged with a different key
  • Be careful about creating applications that are a "suite" and share the same signing key. When apps are part of the same suite, they should share the same linking information in the registry because this linking information will not be removed from the registry until all channels in the suite are removed.
  • It is critical to save the password and keep track of the key that signed an application. If the password is lost, you will not be able to sign any new packages (including new versions) with the same key.
  • Multiple developers can use the same key to sign their applications by "Rekeying" the Roku Streaming Player with a package containing the key and entering the corresponding saved password.

Content Security Model

The SDK supports a content security model using time-expiring random urls and a meta-data feed accessed with SSL mutual authentication. The success of the security model requires a proper implementation by the developer. Roku is not responsible or liable for any losses resulting from the use of this SDK.

The general idea is that a developer could protect the meta-data feed with mutual SSL authentication. Each Roku box has a unique client certificate that is signed by Roku as the Certificate Authority. The firmware also supports a special header, x-roku-reserved-dev-id, that always sends the developer id of the currently running application. Since this is authoritative and can't be forged, the developer's web server can only accept connections from a Roku Streaming Player running their application. Likewise, the Roku Streaming Player can enforce that it is talking to the developer's web server by including the CA certificate of the authority that issued their web server certificate in their channel application. The implementation details of this mutual SSL authentication method are included in the Component Reference Guide.

In the secure meta-data feed, the developer can send the URLs to his content. Since these URLs are long random strings, it would be very difficult for a hacker to guess the URL. Since they are time-expiring, even if they guessed correctly, the damage would be limited as they will disappear when they expire.

Playback of content over SSL-encrypted HTTPS connections ensures that the content is encrypted while en route to the Roky player. The decrypted content is only available for playback and never stored on the Roku Streaming Player. The output ports are also protected: The HDMI port features HDCP and the component video outputs support both CGMS-A and Macrovision. .The USB is not for general purpose use.

Generating Screenshots (Since v3.1 on Roku 1 and v4.3 on Roku 2)

You can generate screenshots of your side-loaded developer channel using the Utilities page. This is done on the "Package Utilities" page of the Developer Web Page on your Roku Streaming Player. Just navigate your side-loaded application to the screen you'd like to capture, then navigate your browser to the Package Utilities page and click on the "Screenshot" button. The image of your application's screen will load in your browser. This will capture only the currently displayed screen in a side-loaded application. It cannot capture images of the home screen or other Roku menus, or images from an application loaded from the Channel Store.

Once the screenshot is captured, you can right-click (on Windows) or control-click (on MacOS) on the image to save it.
