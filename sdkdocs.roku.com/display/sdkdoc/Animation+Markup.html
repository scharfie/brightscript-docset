Node Class References: Animation, Vector2DFieldInterpolator, FloatFieldInterpolator, ColorFieldInterpolator

The following XML Scene Graph markup examples demonstrate how to animate renderable nodes. This allows you to move text, shapes, pictures, and videos around the display screen, fade them in and/or out of view, slide them completely off the display screen area, move them over each other, and change their color.

All of these animation effects are accomplished using Animation node classes that target specific fields of renderable nodes, changing them over a specified time to achieve the animation effect. Depending on the fields to animate, you will use one of three "interpolator" node classes:

  • Vector2DFieldInterpolator
  • FloatFieldInterpolator
  • ColorFieldInterpolator

Let's look at how to use each of the interpolator node classes to animate Label, Poster, and Rectangle nodes, respectively.  

Vector2DFieldInterpolator Markup

Example Application: LabelAnimation.zip
Node Class Reference: Vector2DFieldInterpolator

In this example, we'll use a Vector2DFieldInterpolator node to animate the translation field of a label node. If you look at the labelanimation.xml file in the example application package, you'll note we are using the same the same Label node definition we used in Renderable Nodes Markup previously:

Label Node to Animate XML Example

For this animation, we want to scroll the text "All The Best Videos!" back and forth along the bottom of the display screen. To make this happen, we define an Animation node below the Label node definition in our XML file, with a child Vector2DFieldInterpolator node:

Label Node Animation XML Example

This might look a little confusing so let's break it down piece by piece:

  • We added IDs for both the Animation node and its Vector2dFieldInterpolator child node, for any possibility that later we might want to target these nodes by other nodes, or BrightScript code:

id="scrollbackAnimation"
...
id = "scrollbackLabel"

  • We set the duration of the animation to 10 seconds:

duration="10"

  • We set the animation to repeat (endlessly) after completing:

repeat="true"

  • We are using the most typical "linear" definition of the easeFunction field, which creates a smooth, even animation:

easeFunction="linear"

  • In the Vector2DFieldInterpolator child node, we have two fields with arrays of values that must have the same number of values:

key="[0.0, 0.5, 1.0]"
keyValue="[ [160.0, 620.0], [620.0, 620.0], [160.0, 620.0] ]"

  • the key field array contains the fractional values of the 10 second duration of the animation where the field value specified by the corresponding keyValue array is set; for this animation there are three key values which basically divide the animation into three parts, a beginning (0.0), a middle (0.5), and an end (1.0)

  • the keyValue field array contains the values of the Label node field that will be set at the corresponding fractional value in the key array; in this case it is the X,Y position of the Label node on the display screen

  • The fieldToInterp field then defines which of the Label node fields to animate; in this case it is the translation field of the bottomLabel node as identified by its ID field:

fieldToInterp="bottomLabel.translation"

Now to start the animation, we have to set the control field of the Animation node object to play using BrightScript in a <script> element. Refer to Video Markup to remind yourself how to use the findNode() function to create a component object reference for a target node, and set the control field of the object.

The following shows the position of the text on the display screen correlated with the duration field time, the key field fractional portion of the animation, and the keyValue position of the text:

key (time)                   0.0 (0 seconds)/1.0 (10 seconds)       0.5 (5 seconds)
keyValue                    160,620                                               620,620

It can take a little while to correlate the definitions of these fields with the movement of the text on the screen, so take some time and watch it scroll back and forth...and back and forth... 

FloatFieldInterpolator Markup

Example Application: PosterAnimation.zip
Node Class Reference: FloatFieldInterpolator

In this example, we'll use a FloatFieldInterpolator node to animate the opacity field of a Poster node, causing it to fade in and out of visibility. We'll go ahead and use the same Poster node definition we used in Renderable Nodes Markup previously. The following shows the XML Animation node and FloatFieldInterpolator child node definitions for this:

Poster Node Animation XML Example

We've kept this example simple by using the same duration and key field definitions as in the previous Vector2DFieldInterpolator example. This means that the animation will again last 10 seconds, and be divided into three parts: a beginning at 0 seconds (key value 0.0), a middle at five seconds (key value 0.5), and an end at 10 seconds (key value 1.0). Note that this causes the channelPoster to gradually appear on the display screen, be fully visible at five seconds, then gradually disappear until it is completely invisible at 10 seconds. And again, because the animation repeat field was set to true, this happens over and over and over again... 

ColorFieldInterpolator Markup

Example Application: RectangleAnimation.zip
Node Class Reference: ColorFieldInterpolator

For this final Animation node interpolator example, we'll change the color of a rectangle. To do this, we must use the ColorFieldInterpolator node, which can change the color field of a target node over the time of the animation specified in the Animation node duration field. Once again, to keep it simple, we'll use the same duration, and key field definitions used in the previous interpolator examples, and use the same Rectangle node markup from Renderable Nodes Markup:

Rectangle Node Animation XML Example

Because we used many of the same field definitions as the previous interpolator examples, it should be clear that the only difference here is that the animation changes the color field of the target bottomRectangle rectangle, so it changes the rectangle color rather than moving or fading it. The color changes from blue (0x0000FFFF) at 0 seconds, to black (actually a very dark shade of blue, 0x00000FFF) at five seconds, and back to blue at 10 seconds.
