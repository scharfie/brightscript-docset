DRM

The Roku OS supports a number of DRM formats for protected video content.  The table below summarizes these DRM options and the content formats that support them.  For protected HLS content, Roku recommends using Adobe DRM.  AES-128 encryption is also supported for HLS, but in this case there is no DRM.  Roku thus recommends Adobe DRM with HLS.

  PlayReady Adobe DRM AES-128
HLS   X X
Smooth X    
DASH X    

To use Adobe DRM with HLS streams, a channel must include the requires_aaxs_drm entry in the manifest file.  See the Manifest File page for details.

Copy Protection

The Roku OS also supports HDCP to provide content copy protection between the Roku player's HDMI port and the connected display.  The version of HDCP supported depends on the specific Roku player model, as detailed in the table below.
