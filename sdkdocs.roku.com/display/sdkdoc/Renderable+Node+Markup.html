Beta Documentation

As described in Scene Graph XML Guide, there are four basic renderable node classes (node classes that draw on the display screen):

  • Rectangle
  • Label
  • Poster

  • Video

So to start the tutorial, let's see how to create and display these four node classes on the screen.

Rectangle Markup 

Example Application: Rectangle.zip
Node Class Reference: Rectangle

The Rectangle node class is probably the simplest node to define using XML markup. A Rectangle node just draws a rectangular shape of a certain size on the display screen at a particular location. Because it is so simple, it's a great node class example to show how XML markup is done.

In Rectangle.zip, we create and show a rectangular shape that we intend to use as a bar near the bottom of the screen:

Rectangle XML Example

The following are the important parts of this example that apply to all Scene Graph component XML markup:

  • Per standard XML practice, the XML declaration prolog is placed at the top:

<?xml version="1.0" encoding="utf-8" ?>

  • The component to be defined and created in the XML file is named, and the class name the component is to be derived from is specified, in an XML element that contains the Scene Graph node tree for the component:

<component name="component_name" extends="class_name" >

<XML_element >
  element_definition
</XML_element>

<children>

<Scene_Graph_node_1  
  node_1_attribute_1="attribute_1_definition"
    ... />

<Scene_Graph_node_2 
  node_2_attribute_1="attribute_1_definition"
  ... />

...

</children>

</component>

In the above example, the component to be created is named rectangleScene (component name = "rectangleScene"), which is derived from the Scene class (extends = "Scene"). The example also includes a <script> element to include BrightScript code to set the user interface focus for the screen display that is created:

<script type="text/brightscript" >
<![CDATA[

sub init()
  m.top.setFocus(true)
end sub

]]>
</script>

This is typical practice for most Scene Graph XML markup, and is included in most of the example applications in this tutorial. See Scene Graph Reference, specifically <script>, for more information on using the <script> element.

Next, there is a <children> element. This is added to make XML schema validation (if used) work via XSD. XSD requires that the order of elements be deterministic, and thus for validation to work, Scene Graph node elements need to be wrapped in an element themselves.

Finally, the Rectangle node itself is defined as follows:

  • The rectangle is given an ID of bottomRectangle, to allow the node to be targeted by non-renderable nodes (such as Animation node classes) and BrightScript code:

 id="bottomRectangle"

  • The color of the rectangle is set to blue:

 color="0x0000FFFF"

  • The width of the rectangle is set to 1280 to cover the full width of an HD display screen:

 width="1280"

  • The height of the rectangle is set to 60:

height="60"

  • The position of the rectangle is set at the X,Y screen coordinates of 0,620 so the rectangular bar will be near the bottom of the HD screen:

translation="[0,620]"

Label Markup

Example Application: Label.zip
Node Class Reference: Label

The Label node class writes text in a specified position on the display screen. The Label.zip example writes "All The Best Videos!" near the bottom left of the display screen. The Label node is defined in the component XML file as follows:

Label XML Example

There's a few things to note about this Label node definition:

  • Again, in this case, we want to define an ID to allow other non-renderable nodes, such as Animation node classes, to target the node:

id="bottomLabel"

  • We define a height of 60 (to set the height explicitly), but a width of 0 (to allow the width to be determined by the width of the specified text):

height="60"
width="0"

  • We define the text to be shown on the display screen:

 text = "All The Best Videos!"

  • We define the position of the text to be aligned to the left of the label area (which in this case, is not really important, since the label area is the width of the text itself, because the width field was set to 0), and centered within the specified height of 60:

horizAlign = "left"
 vertAlign = "center"

  • We define the position of the label to be near the bottom of the HD screen, offset right 160 pixels to the edge of the SD screen "safe zone":

 translation="[160,620]"

Poster Markup

Example Application: Poster.zip
Node Class Reference: Poster

The Poster node class draws a graphic image file in a specified location on the display screen. The Poster.zip example draws the channelbug.png graphic image file included in the application images directory near the bottom right of the display screen:

Poster XML Example

This simple example shows close to the minimum amount of markup required to draw a graphic image file on the screen:

  • Again, if we ever want to target this node with another node class or BrightScript code, we assign an ID for the node:

id="channelbugPoster"

  • Of course, we need to specify the URI path to the graphic image file (this could be any valid URI including a file downloaded from an HTTP server):

uri="pkg:/images/channelbug.png"

  • In this case, we set the width and height of the image to be drawn on the screen to the pixel width and height of the graphic image file itself, which is generally the best way to draw the clearest image on the screen (though the Poster node class does include automatic scaling for image size specifications that do not match the pixel dimensions of the graphic image file, and by default, if width and height field values are set to 0, the image is drawn to the pixel width and height of the graphic image file itself):

width="220"
height="100"

  • As usual, we set the location of the image on the display screen in the translation field, in this case near the bottom right of the display screen, offset to the edge of the "safe zone" of an SD screen:

translation="[900,600]"

And the result on the display screen is:

There's several more fields that could have been set for this image, to control the scale of the image, and the opacity of the image, for example. But for now, we'll just get the image on the screen, and the Poster node class reference has complete information on all the fields that could have been set. However, you may notice that based on the image size and location (not to mention the name of the graphic image file), this Poster node example could be used for a very specific purpose in a real channel application...as are all the previous simple examples we've seen so far.

Video Markup

Example Application: Video.zip
Node Class Reference: Video

It's time to actually play a video. The Video node class let's you play a video using markup similar to what we've used in the previous simple examples. But for this example, we'll add some BrightScript code in the <script> element to set up the video play parameters:

Video XML <script> Element Example

If you are familiar with BrightScript, the above <script> element looks familiar. The usual <script> element init() function seen in previous Scene Graph component XML file examples is there, but with a call to a new function we write, setVideo(). The setVideo() function sets the parameters for playing the video in a ContentNode node, to be assigned to the content field of the target Video node.

First we create a ContentNode node object, using the createObject() function, then we set the URL, video title, and the streaming format for the video in the ContentNode node object. Then the BrightScript code uses the findNode() function to find a node in this (m.top) Scene Graph component node tree with an ID of musicvideos, and assigns it to the component object reference m.video.

From there, the BrightScript code can define the Video node object fields, setting the content field to the videoContent ContentNode node object, and the control field to play, which will start the video immediately after the Video node object is created from the Scene Graph node tree. Note that it is required to set the control field of a Video node (and other similar types of node classes that can begin and end operations) in BrightScript. And generally, you will want to start playback operation of a Video node in response to a user input event, which must be handled using BrightScript in a <script> element.

The Video node markup itself is simple, and is just like the previous simple examples for setting the Rectangle, Label, and Poster node sizes and positions on the display screen:

Video XML Example

Note that we must set the ID of this Video node to the same name used as a parameter in the findNode() BrightScript call in the <script> element above:

id="musicvideos"

Because we want the video to play in the entire HD screen area, we set the video size to the full HD screen area, and locate it at the top left of the screen:

width="1280"
height="720"
translation="[0,0]"
